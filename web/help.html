Avadis iManage Web Client

Release Date: January 21, 2016

Preferred Browser: Chrome

Avadis iManage is an enterprise-wide data and analysis management platform for Bio-Imaging researchers.
It is designed to let users manage, visualise and annotate and share complex imaging records generated by modern automated microscopes.
This document is provided to help the first-time user start using Avadis iManage as quickly as possible.

Users need one of the following to interact with Avadis iManage:

iManage Web Client

iManage Web Client is browser based application to access, annotate and analyse image records from iManage server.

User interface for iManage Web Client consists of five main areas:
  1. Navigation and selection pane
  2. Thumbnail gallery
  3. List of records (Summary table)
  4. Image View with Image Controls (Sites, Channels, Contrast, Overlays, Comments and metadata tables)
  5. Workflows with analysis links.
When a user selects one of the virtual folders in the Navigator pane, the corresponding records are displayed as a list in the Summary Table.
Thumbnail gallery shows the exact replica of the Summary Table contents.
If user clicks on a record either in the Summary Table or in the Thumbnail Gallery it is displayed in the Image View.

iManage Acquisition Client

iManage Acquisition Client is a desktop application for viewing and annotating data locally before uploading to the central server.
It can be downloaded from here. Acquisition Client
User interface for iManage Acquisition Client consists of three main areas:
  1. Thumbnail gallery
  2. List of records (Summary table)
  3. Image View with Image Controls (Sites, Channels, Contrast, Overlays, Comments and metadata tables)
If user clicks on a record either in the Summary Table or in the Thumbnail Gallery it is displayed in the Image View.

iManage Admin Client

iManage Admin Client is a browser based application to facilitate user management, resource management and access control.
iManage Admin Client user interface contains multiple tabs to view and edit the users, resources and other information present in the system.

Features

iManage Access Control

  • Avadis iManage uses oAuth based authentication to provide secure access to the data.
  • Users can login to the system using login-password created and stored in iManage system.
  • Alternately, iManage allows integration with LDAP directories to support enterprise wide central login-passwords.
  • A "group of users" are organised in "Projects".
  • Each user can belong to multiple project and each project can have multiple users.
  • Each user has a specific access permission, namely READ, WRITE, UPLOAD, DELETE in each project.
  • All users within the same "Project" can share and view all the records in the "Project".
  • In addition, users can Share or Transfer records to other projects.

iManage Data Model

  • Avadis iManage uploads data in original format (no coversion) to the storage location specified in the configuration.
  • This data can be seamlessly browsed and accessed via the file-system browsers.
  • Data is organised in "Projects".
  • Each project has a specific "Storage Quota".
  • Certain meta-data fields like "No. of Channels", "No. of Slices", "No. of Frames" etc. are extracted from the source image files and populated in a database for accessing and visualising image data-sets as 5D image records.
  • Some image level meta-data is also extracted and populated in database.
  • These meta data fields are available for all records in iManage.
  • In addition users can add name-value pairs, file attachments, comments and visual objects to the records.
  • This user added data is also shared with all the users in the Project.

iManage Data Import

  • iManage Acquisition Client can be used to import and inspect records locally before uploading to the server.
  • First of all user needs to select the Project to which he intends to upload the record.
  • Using the Import -> Import File(s) menu, user can specify the directory to import.
  • All image files in the specified directory are considered for import.
  • Image files are imported using Loci Bio-formats.
  • Following formats are supported.
  • A record is created for each series in the source file.
  • If dimensions match, users can merge multiple records created from same source files.
  • Import -> Import Using Filename(s) menu can be used to create records with meta-data embedded in the filenames.
  • Status of import is shown in a console window.
  • Imported records are shown in summary table and thumbnails.
  • User can browse records and add attachments, comments, text and visual annotations of imported records.
  • User can also update channel colour and contrast settings.
  • Duplicate records(records already uploaded on server) appear as read-only records.

iManage Data Upload

  • Records imported in iManage Acquisition Client can be uploaded to the iManage Server.
  • User can select the records to be uploaded.
  • If the Project has available quota files in original format are uploaded to the storage.
  • Meta-data containing annotations, comments, contrast settings, channel colour etc. are uploaded to the server.
  • All members of the project can view the uploaded records using iManage Web Client.
  • Status of each upload can be monitored via the iManage Acquisition Client.
  • Users can install iManage Upload Service to upload records in background even after they quit the iManage Acquisition Client.
